# ApricotPlayer 0.9.53

This build restores the old fast-seek behaviour for YouTube playback.

## Fixes

- YouTube playback now prefers a small progressive MP4 stream with audio+video before falling back to audio-only m4a. This matches the pre-refactor behaviour where mpv can jump far forward immediately after playback starts.
- The stream URL cache key now includes the playback format profile, so old cached audio-only stream URLs from 0.9.45-0.9.52 are not reused.

## Verification

- Confirmed the new selector chooses YouTube format `18` MP4 360p on a test video, while the previous audio-first selector chose format `140` m4a.
- `python -m compileall -q wx_main.py main.py apricot`
- `python -m ruff check --select E9,F63,F7,F82 wx_main.py main.py apricot`
