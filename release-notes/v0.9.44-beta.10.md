# ApricotPlayer v0.9.44 Beta 10

Tenth beta in the 0.9.44 cycle. Fixes a seek regression introduced in beta.7 that caused a 2–5 second stall every time the user seeked forward or backward in a stream.

## Fixes

- **Seek stall fixed.** Seeking forward or backward (arrow keys, large seek, etc.) now responds instantly, as it did before the modular refactor. The regression was introduced in beta.7 when `--stream-lavf-o=reconnect_streamed=1` was added to mpv's launch arguments. This option tells ffmpeg to tear down and reconnect the HTTP connection whenever a seek is issued on a "streaming" (non-seekable) URL — which includes every YouTube, SoundCloud, and other web stream. The result was a 2–5 second network round-trip on every single seek press before playback resumed at the new position. The option is now removed. `reconnect=1` and `reconnect_delay_max=5` are retained so mpv still retries after an actual network drop on seekable streams.
