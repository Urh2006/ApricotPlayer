# ApricotPlayer v0.9.44 Beta 9

Ninth beta in the 0.9.44 cycle. Adds native JAWS speech support so that all player announcements — track started, volume changed, seek, speed, repeat, shuffle, etc. — are spoken by JAWS with the same reliability as NVDA.

## New features

- **JAWS screen reader support.** All player announcements that previously reached only NVDA (via its controller DLL) now also reach JAWS via its COM automation server (`FreedomSci.JawsApi`). The implementation uses only Windows `ctypes` — no additional Python packages are required and nothing needs to be installed. It talks directly to a running JAWS process through its IDispatch interface, calling `SayString(text, True)` (interrupt-and-speak) on each announcement.

  JAWS is tried only when NVDA has not already handled the text, so systems where both screen readers are running simultaneously do not get duplicate speech. On machines where JAWS is not installed the ProgID lookup (`CLSIDFromProgID`) is attempted once at first use, returns nothing, and all subsequent calls return immediately — zero runtime overhead.

  `EVENT_SYSTEM_ALERT` continues to be suppressed when either NVDA or JAWS handles the text directly (preventing double-announcement). `EVENT_OBJECT_NAMECHANGE` and `EVENT_OBJECT_VALUECHANGE` are always fired for Narrator and any other IAccessible-based screen reader.
