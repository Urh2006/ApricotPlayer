# ApricotPlayer v0.9.44 Beta 12

Twelfth beta in the 0.9.44 cycle. Fixes JAWS not announcing player shortcuts when focus is in the results list, and prevents crashes caused by unhandled exceptions in keyboard event handlers.

## Bug fixes

- **JAWS now announces F2, F3, V and other player shortcuts while browsing results.** Two related bugs combined to silence these keys entirely when focus was in the results list: `player_shortcuts_allowed` incorrectly returned `False` for that focus location (it should return `True`, because player shortcuts are valid there), and a stray `return True` in `handle_player_shortcut_event` consumed non-navigation keys without executing their actions. Both are fixed — JAWS will now speak the seek position, volume, and track changes triggered by these keys the same way it does elsewhere.

- **Fixed crashes when pressing keys in the results list.** The `on_results_key` event handler had no exception guard. Any unhandled error raised by actions such as downloading audio, playing the selected item, or opening a context menu propagated to wxPython's main loop and terminated the app. The handler is now wrapped in `try/except` so a single failed action is silently absorbed rather than crashing the program.
