# ApricotPlayer 1.0.0 Beta 8

This beta is a targeted audio handoff hotfix for the reported startup burst/crackle when changing tracks, especially with Volume boost involved.

## Fixes
- Silenced the previous mpv process before replacing playback, so a boosted old player cannot leak a loud frame while the next item is starting.
- Kept session volume intact by remembering it before the temporary stop-time mute/volume-zero handoff.
- Invalidated stale startup/volume workers earlier during player stop, so old workers cannot unmute or adjust a player that is already being replaced.

## Notes
- The older diagnostic report showed Volume boost disabled at that exact moment, but the new tester description matches a boosted handoff/startup-state leak. This build fixes that path without changing the EQ profile math or the normal volume controls.
- Local installs are not updated automatically from this build unless you choose the beta channel/update manually.
