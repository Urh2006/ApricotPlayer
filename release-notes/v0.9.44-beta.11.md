# ApricotPlayer v0.9.44 Beta 11

Eleventh beta in the 0.9.44 cycle. Makes the stream URL resolver cache persistent across program restarts.

## Improvements

- **Persistent stream URL cache.** The URL resolver cache was in-memory only: closing and reopening the app wiped it completely, so the "instant second play" speed-up only applied within a single session. Now the cache is saved to `%APPDATA%\ApricotPlayer\stream_url_cache.json` when the app exits and reloaded on the next startup. Expired entries (based on each entry's `expires_at` timestamp, which is capped to the URL's own `expire=` parameter for YouTube links) are filtered out at load time, so stale entries never reach a running session. Corrupt or missing cache files are silently ignored — the cache is a pure speed optimisation and its loss is never fatal.
