## What's new in 0.3.5

- The Windows installer now installs a PyInstaller one-folder build instead of the portable one-file executable.
- Installed launches are faster because ApricotPlayer no longer needs to unpack the full one-file archive on every start.
- The portable `ApricotPlayer.exe` is still included in the release for quick testing and sharing.
- Installer updates clean the old runtime folder before installing the new one.
- Build scripts now support both packaging modes: `onefile` for portable builds and `onedir` for the installer.
