# ApricotPlayer v0.9.44 Beta 6

Sixth beta in the 0.9.44 cycle. Targets two regressions introduced during the modular refactor that beta testers identified as slowness and key-handling misbehaviour: a wrong mpv cache option name and unnecessary pre-checks in the global keyboard hook.

## Bug fixes

- **Seek performance and back-buffer cache restored.** `start_mpv` was passing `--demuxer-max-back-bytes` to mpv, but that option does not exist — the correct name is `--demuxer-back-bytes`. With an unrecognised option mpv silently ignores it, so the back-buffer size was effectively zero: every seek backward had to re-download data from the network instead of reading from the on-disk cache. The correct option name is now used; seeking back in a track returns to native performance.

- **Keyboard lag and misbehaving shortcuts fixed.** Two unnecessary pre-checks were added to `on_char_hook` during the modular refactor that were not present in the original codebase.

  - A Tab-key special-case block was skipping the Tab event immediately in non-player contexts before the normal Tab-navigation handlers (`handle_background_player_tab_navigation`, `handle_player_tab_navigation`) had a chance to run. This caused Tab to behave differently depending on which screen was active.

  - An `activate_focused_button_from_key` call was consuming Enter/Space keystrokes on focused buttons by programmatically firing the handler — in addition to the button's own `EVT_BUTTON` binding that fires via native wx processing. On buttons that bind both paths, this produced double-activation and swallowed the native key event, so the button's visual press-feedback was lost.

  Both pre-checks are removed. The keyboard hook now matches the pre-refactor logic: modifier-only events and shortcut-capture controls are filtered first, then native widget pass-throughs (Choice/ComboBox, CheckBox/Slider/SpinCtrl, text controls) run as before, followed by the full shortcut matching chain.
