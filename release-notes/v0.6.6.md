# What's new in version 0.6.6

- Fixed the installer updater relaunch path so ApricotPlayer opens the newly installed version immediately after updating.
- Removed the installer restart-applications flag from ApricotPlayer's updater flow, preventing the old version from reopening during setup.
- After setup finishes, the updater now verifies the installed executable location, stops stale ApricotPlayer processes in known install folders, and launches the verified new executable.
- Silent installer updates now launch the updated app at the end of setup, with duplicate-launch suppression so older updater scripts do not show an extra already-open dialog.
