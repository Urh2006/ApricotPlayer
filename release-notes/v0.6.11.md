# What's new in version 0.6.11

- Rebuilt browser cookie handling so playback and downloads use ApricotPlayer's cached `cookies.txt` instead of repeatedly reading locked Brave/Chrome/Edge databases.
- Added automatic cookie repair: when YouTube asks for sign-in or bot confirmation, ApricotPlayer refreshes cookies from the selected browser and retries the same playback or download.
- Added browser profile discovery for Brave, Chrome, Edge, Chromium, Vivaldi, Opera, and Firefox.
- Added a Browser profile setting with an Auto option that tries discovered profiles and chooses the one with usable YouTube/Google cookies.
- Cookie export errors now include profile-level details instead of only saying `failed to load cookies`.
- Clarified that `Choose cookies.txt file` is for an exported Netscape-format cookies file, not Brave's internal encrypted cookie database.
