# ApricotPlayer 0.9.57

Hotfix release for player-session audio settings.

## Fixes
- Fixed player session persistence so changing to another video without closing the player keeps the volume chosen by the user, including results, channel videos, playlists, queue playback, and background playback.
- Tightened the reset path used when the player is actually closed: Escape/Close player now returns session-only audio state to defaults for the next player session.
- Output device, bass boost, volume boost, repeat, autoplay-next session state, shuffle, and EQ session state now follow the same rule: keep them while the player session continues, reset them when the player is closed.
