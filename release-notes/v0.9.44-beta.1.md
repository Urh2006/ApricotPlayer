# ApricotPlayer v0.9.44 Beta 1

This is the first beta release in the 0.9.44 cycle. It contains a performance pass and a handful of bug fixes. No new features — the goal was to make the existing code faster and more correct.

## Performance improvements
- **Regex compilation overhead eliminated.** Several functions that are called on every result, every sort, and every download were recompiling their regex patterns from scratch on each call. All hot-path patterns are now compiled once at module load time. The most noticeable change is faster rendering when browsing long result lists, loading RSS feeds, or sorting a local folder with many files.
- **Startup file parse time reduced.** Around 500 blank lines left over from the modular refactor were removed from `wx_main.py`. Python's parser processes every line, so this trims a small but real chunk of startup overhead.

## Bug fixes
- Fixed a file handle leak when the mpv player failed to start. If subprocess launch raised an exception after the log file had already been opened, the handle was silently left open until the next playback attempt.
- The GitHub release publisher now correctly marks beta/alpha/rc releases as pre-releases on GitHub so they appear in the beta update channel but not in the stable channel.
