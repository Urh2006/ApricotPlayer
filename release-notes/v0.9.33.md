# ApricotPlayer v0.9.33

## Fixes
- Fixed arrow key navigation on combo boxes. Pressing Up/Down while a drop-down control had focus was silently swallowed by the global key hook before Windows could pass the event to the control. Native arrow, Home, End, and keyboard-search behaviour is now fully restored on all drop-down controls throughout the app.
- Fixed yt-dlp reload after a component update. After a self-contained yt-dlp update the player was not always switching to the new version because the cached module reference was being reset in the wrong namespace. The new version is now guaranteed to be picked up on the next use.
- Fixed the Update Channel setting being silently discarded when loading settings if the stored value was unrecognised. It now correctly falls back to "Stable".
- Added missing Update Channel labels to all 25 non-English languages (Arabic, Czech, German, Greek, Spanish, Finnish, French, Hindi, Croatian, Hungarian, Indonesian, Italian, Japanese, Korean, Dutch, Polish, Portuguese, Romanian, Russian, Slovak, Serbian, Swedish, Turkish, Ukrainian, Chinese). The Settings screen showed an empty label for this control in every language except English and Slovenian.
- Removed a redundant `import re` statement buried inside the lyrics display function. Internal cleanup with no visible effect.
