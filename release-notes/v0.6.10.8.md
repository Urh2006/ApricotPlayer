# What's new in version 0.6.10.8

- Improved browser cookie export for YouTube sign-in / bot-confirmation problems.
- When the selected browser is open, ApricotPlayer now asks whether to close it before exporting cookies, which helps avoid the Chrome locked cookie database error.
- Added a Settings button to choose a `cookies.txt` file if you already have one.
- Playback and downloads now automatically reuse ApricotPlayer's cached exported `cookies.txt` file when no explicit cookies file is configured.
- Error messages for locked browser cookie databases now point users to the built-in export flow instead of leaving them stuck.
