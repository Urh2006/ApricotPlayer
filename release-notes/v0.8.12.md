# What's new in version 0.8.12

- Manual cookies files from browser extensions are now imported instead of used blindly: ApricotPlayer normalizes Netscape/Mozilla `cookies.txt`, converts common JSON exports, and accepts simple `Cookie:` header exports.
- Existing manually selected cookie files are repaired automatically the next time cookies are needed, so testers do not have to reselect their file after updating.
- Added an optional Browser User-Agent setting for cookies files for sites that require matching browser headers together with cookies.
- Cookie diagnostics now keep usable cookies but warn clearly when the selected file does not contain YouTube/Google login cookies.
