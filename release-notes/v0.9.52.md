# ApricotPlayer 0.9.52

This build focuses on the tiny but noticeable lag that could be felt around player transitions after the modular refactor.

## Fixes

- History recording no longer writes the history JSON file before the player page is shown. It updates memory immediately and writes in the background.
- Resume-position saving now uses short mpv IPC timeouts and reuses known duration metadata where possible, so a slow mpv response cannot hold the UI for long during track switches.
- Replacing playback now waits less aggressively for the old mpv process to exit before continuing.
- Stream URLs now skip unnecessary local-file path checks.
- App shutdown performs one final synchronous history save so the faster background history path does not drop recent entries.
