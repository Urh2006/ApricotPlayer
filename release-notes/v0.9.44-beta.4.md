# ApricotPlayer v0.9.44 Beta 4

Fourth beta in the 0.9.44 cycle. Fixes the updater silently reporting "up to date" for all existing beta testers and a regression where the installer would overwrite a custom install location on update.

## Bug fixes

- **In-app updater now offers beta updates by default.** The update channel defaulted to "stable", which meant beta releases were never surfaced through the in-app updater — the app would check for updates, find none (because no stable release exists yet), and report "already up to date." The default is now "beta". Existing users who already have "stable" stored in their settings are migrated automatically on first launch of this build, so no manual settings change is required.

- **Installer no longer resets the install directory on update.** `UsePreviousAppDir` was set to `no`, which caused the installer to default back to `C:\Program Files\ApricotPlayer` even when the previous version had been installed elsewhere. Updating via the in-app updater or by running the installer directly would install the new version to a different path than the one the user's shortcuts pointed to, so the old version kept launching. The installer now respects the previously recorded install directory (`UsePreviousAppDir=yes`).
