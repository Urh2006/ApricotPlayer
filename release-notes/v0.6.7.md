# What's new in version 0.6.7

- Fixed single video downloads so they always stay in single-video mode instead of accidentally following playlist URLs when playlist ordering is enabled.
- Dynamic search metadata hydration now covers newly loaded result pages, so later batches can also update from Uploaded unknown to Uploaded X ago.
- Reused one yt-dlp session while hydrating a visible metadata batch, making result metadata updates lighter.
- Fixed search result metadata updates so NVDA focus no longer jumps from result to result while upload ages are being filled in.
- Throttled duplicate download progress UI events so large downloads and batch downloads feel smoother.
- Subscription checks and RSS refreshes now continue when one channel or feed fails, preserving the rest of the update.
- Cached the HTTPS certificate context used for GitHub, podcast, RSS, and update requests.
- Cleaned up static-analysis warnings in the Python sources.
